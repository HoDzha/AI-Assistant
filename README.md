# Freelance Flow AI

AI-ассистент для управления задачами фрилансера: FastAPI backend, статический frontend и LLM-анализ рабочей загрузки.

## Что умеет

- хранить задачи в SQLite через отдельный слой хранилища с возможностью заменить движок БД;
- создавать, редактировать, дублировать, архивировать и фильтровать задачи;
- анализировать активные задачи через OpenAI API и строить приоритеты, дневной план и сводку по проектам;
- работать с fallback-логикой, если ответ модели неполный;
- экспортировать результаты анализа и быстро загружать демо-набор данных.

## Стек

- `backend/app` - FastAPI, бизнес-логика, модели, OpenAI-клиент и storage-абстракции;
- `frontend` - HTML/CSS/JavaScript интерфейс без сборщика;
- `tests` - интеграционные и модульные тесты;
- `scripts` - служебные скрипты для локальной подготовки данных.

## Быстрый старт

1. Создай локальный `.env` на основе `.env.example`.
2. Установи зависимости:

```bash
python -m pip install -e .[dev]
```

3. Запусти backend:

```bash
uvicorn app.main:app --app-dir backend --reload
```

4. Запусти frontend:

```bash
python -m http.server 5500 --directory frontend
```

5. Открой `http://127.0.0.1:5500`.

## Переменные окружения

- `OPENAI_API_KEY` - ключ OpenAI API;
- `OPENAI_BASE_URL` - базовый URL API;
- `OPENAI_MODEL` - модель для анализа задач;
- `DATABASE_URL` - строка подключения к БД;
- `LOG_LEVEL` - уровень логирования backend.

## Проверка проекта

```bash
python -m pytest

```

## Перед публикацией на GitHub

- не коммить локальный `.env` и ключи доступа;
- не пушь `.venv`, логи, локальную БД и build-артефакты;
- при необходимости очисти демо-данные из локальной базы или заново заполни её через скрипты;
- добавь лицензию и скриншоты интерфейса, если хочешь оформить репозиторий как портфолио-проект.
